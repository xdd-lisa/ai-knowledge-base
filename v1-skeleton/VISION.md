# AI Knowledge Base — 项目愿景文档

## 一句话概述

构建一个**全自动 AI 知识情报系统**，每日从 GitHub Trending、Hacker News、arXiv 等源头抓取 AI 领域最新动态，通过多 Agent 协作完成采集→分析→整理→发布，生成高质量的中文 AI 技术日报 / 周报。

---

## 一、我们要解决什么问题

AI 领域信息爆炸，每天有数百篇论文、数千个开源项目、无数讨论。一个 AI 工程师 / 研究员每天花 2-3 小时筛选信息，仍然会漏掉重要内容。

**核心痛点：**

| 痛点 | 现状 |
|------|------|
| 信息过载 | 源头太多，不知道该看什么 |
| 语言壁垒 | 优质内容以英文为主，中文开发者获取效率低 |
| 价值判断 | 无法快速判断一篇论文 / 项目是否值得深读 |
| 持续性 | 靠人工整理不持久，三天打鱼两天晒网 |

**我们提供的价值：** 每天 10 分钟，让用户掌握 AI 领域最重要的 20 件事。

---

## 二、产品形态

### 输出物

- **AI 技术日报**（Markdown / 网页 / 公众号）：每日自动发布，包含 Top 10-20 条精选内容，每条附一句话摘要 + 为什么重要 + 推荐度打分
- **AI 技术周报**：每周汇总趋势，提供深度分析
- **知识库检索**：所有历史内容可全文检索，构建个人 / 团队的 AI 知识图谱

### 输入源（Phase 1）

| 数据源 | 内容类型 | 更新频率 |
|--------|----------|----------|
| GitHub Trending | 开源项目、工具、框架 | 每日 |
| Hacker News | 技术讨论、观点、Show HN | 实时 |
| arXiv (cs.AI, cs.CL, cs.CV, cs.LG) | 学术论文 | 每日 |
| Twitter / X (Phase 2) | AI 领域 KOL 动态 | 实时 |
| Reddit r/MachineLearning (Phase 2) | 社区讨论 | 每日 |

## 三、Agent 协作架构

系统采用**多 Agent 流水线**，每个 Agent 只做一件事，做好后交给下一个：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 采集 Agent │───▶│ 分析 Agent │───▶│ 整理 Agent │───▶│ 发布 Agent │
│ Collector  │    │ Analyst   │    │ Curator   │    │ Publisher │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
      │               │               │               │
   原始数据        分析结果         排版内容         发布上线
```

### Agent 1: 采集 Agent（Collector）

**职责：** 从各数据源抓取原始数据，做初步去重和过滤。

- 调用 GitHub API 获取 Trending 仓库（语言过滤：Python / JS / TS / Rust / Jupyter Notebook）
- 调用 Hacker News API 获取热门帖子（关键词过滤：AI / LLM / GPT / ML / DL）
- 调用 arXiv API 获取最新论文（分类：cs.AI / cs.CL / cs.CV / cs.LG）
- 将原始数据写入中间存储，标记来源和时间戳
- 对跨源重复内容（如同一论文在不同源头出现）进行去重

### Agent 2: 分析 Agent（Analyst）

**职责：** 对每条内容进行理解、评估、标注。

- **内容理解：** 读取 README / 摘要 / 正文，用 LLM 生成一句话中文摘要
- **价值打分：** 基于 stars 增速、讨论热度、作者声誉、创新程度，输出 1-5 星推荐度
- **标签分类：** 自动打标签（LLM / Agent / RAG / 多模态 / 推理 / 训练 / 部署 /  benchmark / 工具）
- **关联分析：** 识别与历史内容的关联（同一项目的新版本、同一方向的多篇论文）
- **趋势识别：** 当某方向连续多天出现高热度内容时，标记为"趋势"

### Agent 3: 整理 Agent（Curator）

**职责：** 将分析结果组织成可读性强的内容。

- **排序 + 筛选：** 综合热度、打分、多样性，选出每日 Top 15-20
- **生成文案：** 为每条内容生成标题、一句话摘要、"为什么值得关注"
- **格式排版：** 输出 Markdown 格式，支持网页和公众号两套模板
- **周报汇总：** 每周日额外生成一份周报，包含本周趋势回顾和图表

### Agent 4: 发布 Agent（Publisher）

**职责：** 将整理好的内容发布到各渠道。

- 写入知识库数据库（用于检索和存档）
- 推送到 GitHub Pages / 自有站点
- 推送到钉钉 / 飞书 / Slack（适合团队订阅）
- 更新 RSS Feed
- Phase 2: 公众号草稿箱自动写入

## 四、技术原则

1. **简单优先：** 第一版用脚本 + LLM API 跑通全流程，不引入重型框架。每个 Agent 就是一个独立的 Python / TypeScript 脚本。
2. **可观测：** 每一步有日志、有留存、有回放。出问题了能定位到具体 Agent。
3. **成本可控：** LLM 调用用量化模型（如 DeepSeek），每条内容分析成本控制在 ¥0.01 以内。
4. **人工兜底：** 完整自动化，但提供审核入口——整理 Agent 的输出在发布前可选人工 review。
5. **数据积累：** 从 Day 1 就把所有中间数据存下来，未来做趋势分析和知识图谱。

## 五、Phase 规划

### Phase 1: MVP（2-4 周）

- [ ] 采集 Agent：GitHub Trending + Hacker News + arXiv 三条管道
- [ ] 分析 Agent：LLM 摘要 + 打分 + 打标签
- [ ] 整理 Agent：生成 Markdown 日报
- [ ] 发布 Agent：输出到本地文件 + 推送到 Git
- [ ] 定时调度：每天自动运行一次（GitHub Actions / cron）

### Phase 2: 增强体验（1-2 月）

- [ ] Web 前端：日报浏览 + 搜索 + 历史
- [ ] 新增数据源：Twitter / X、Reddit
- [ ] 多团队订阅分发（钉钉机器人 / 飞书机器人）
- [ ] 周报自动生成 + 趋势图表

### Phase 3: 知识图谱（3-6 月）

- [ ] 实体提取与关联（论文 ⇄ 代码仓库 ⇄ 作者 ⇄ 机构）
- [ ] 个性化推荐：用户订阅特定方向，只推送相关内容
- [ ] 知识问答：用户可以用自然语言问"最近 RAG 有什么进展？"

## 六、成功指标

| 指标 | 目标（Phase 1 结束） |
|------|----------------------|
| 数据源覆盖 | 3 个（GitHub / HN / arXiv） |
| 日报覆盖率 | ≥ 90% 的天数按时产出 |
| 单日条目 | 15-20 条精选 |
| 端到端耗时 | ≤ 30 分钟（从采集到发布） |
| 单日成本 | ≤ ¥5（LLM API 调用） |
| 摘要质量 | 人工抽检正确率 ≥ 85% |
| 核心用户 | ≥ 10 位日常读者 |

## 七、类比定位

| 产品 | 我们的差异化 |
|------|-------------|
| **Hacker News** | 我们做精选 + 中文摘要 + 打分，而非原始列表 |
| **Papers With Code** | 我们覆盖更广（不仅是论文+代码），且是推送而非检索 |
| **即刻 / 知乎热榜** | 我们只做 AI 这一个垂类，比泛科技更深 |
| **竹白 / Substack  Newsletter** | 我们是全自动 Agent 驱动，不是人工编写 |

简单说：**面向中文开发者的、Agent 驱动的、AI 垂直情报服务。**
